# AGENTS.md

## 项目定位

- 本项目是智慧农业大棚相关工程，包含 STM32、MX6U、文档、测试和网页端内容。
- 默认使用中文交流、说明问题和编写新增文档；如已有文件明确使用其他语言，遵循原文件风格。

## 工作原则

- 修改前先阅读相关目录下的文档，优先查看 `文档/`、`MX6U/`、`docs/` 中与任务直接相关的说明。
- 保持改动最小化，只处理当前任务所需内容，不做无关重构。
- 不要删除、覆盖或回退用户已有改动，除非用户明确要求。
- 涉及路径、命令、配置时，优先给出可直接执行或可直接定位的信息。

## 嵌入式与工程文件

- 涉及 STM32、Keil、CubeMX 工程时，注意代码与工程配置保持一致。
- 如果新增源文件、驱动、中间件或 BSP，需同步检查 Keil 工程和 CubeMX 相关元数据是否需要更新。
- 对自动生成代码保持谨慎，除非确认修改位置可长期保留，否则优先放在用户代码区或项目约定的可维护位置。

## 开发板操作

- 开发板默认通过 SSH 访问，目标板地址以项目文档为准。
- 当前已确认的开发板登录方式参考 `文档/网络配置.md`。
- 进行远程操作时，优先采用非破坏性命令；删除、覆盖、批量修改前先确认目标路径和影响范围。
- 涉及运行程序、修改配置、传输文件时，说明实际执行内容和目标位置。
- 所有与虚拟机和开发板相关的操作，视情况同步记录到 `文档/任务文档/虚拟机与开发板.md`，包括地址变更、登录方式、目录位置、设备验证结果和关键配置调整。

## NFS 挂载

- 当前已打通的 NFS 链路为：虚拟机 `192.168.137.3` -> 开发板 `192.168.137.106`。
- 虚拟机实际导出目录为 `/home/lan/linux/nfs`，不是 `/home/lan/linux`。
- 开发板挂载点为 `/mnt/nfs`。
- 开发板验证通过的挂载命令为 `mount -t nfs -o vers=3,nolock,proto=tcp 192.168.137.3:/home/lan/linux/nfs /mnt/nfs`。
- 开发板 `/etc/fstab` 已写入 `192.168.137.3:/home/lan/linux/nfs /mnt/nfs nfs vers=3,nolock,proto=tcp 0 0`，可通过 `mount -a` 重新挂载验证。
- 需要共享文件给开发板时，优先放入虚拟机 `/home/lan/linux/nfs`，再从开发板 `/mnt/nfs` 访问。

## 文档与验证

- 修改配置、脚本或行为后，尽量补充必要说明到相应文档，而不是仅在对话中描述。
- 声称“已完成”“已修复”前，应尽量执行对应验证步骤；如果未验证，需明确说明。

## 禁止事项

- 不要随意执行破坏性命令，如批量删除、强制覆盖、硬重置。
- 不要假设开发板、虚拟机、主机的账号密码和用途完全相同，应以文档和实际验证结果为准。
- 不要在未确认的情况下改动与当前任务无关的网页、测试或工具链文件。

你现在的 Windows 以太网已经被 ICS 设置为：

192.168.137.1
255.255.255.0

因此整个内部网络必须使用 **192.168.137.0/24 网段**。不要再使用其他网段。

网络结构应当这样设计：

WiFi（外网 10.22.x.x）
↓
Windows NAT
192.168.137.1
↓
以太网
↓
开发板 + 虚拟机

具体配置如下。

Windows（你现在已经是这个）
IP：192.168.137.1
子网掩码：255.255.255.0

开发板设置：

IP：192.168.137.2
子网掩码：255.255.255.0
网关：192.168.137.1
DNS：192.168.137.1 或 8.8.8.8

虚拟机设置：

VMware 网络模式：
桥接（Bridge）→ 指定 **以太网网卡**

Ubuntu 网络：

IP：192.168.137.3
子网掩码：255.255.255.0
网关：192.168.137.1
DNS：8.8.8.8 或 192.168.137.1

这样形成：

Windows
192.168.137.1

开发板
192.168.137.2

虚拟机
192.168.137.3

验证步骤：

在虚拟机：

ping 192.168.137.1
ping 192.168.137.2
ping 8.8.8.8

如果都通，则说明：

虚拟机 ↔ 开发板 通
虚拟机 ↔ Windows 通
虚拟机 → 外网 也通（通过 Windows NAT）

再提醒两个常见错误：

第一，VMware 桥接必须指定 **以太网网卡**，不要用 Automatic，否则可能桥接到 WiFi。

第二，开发板和虚拟机必须是 **192.168.137.x**，否则 ICS 网络不会转发。

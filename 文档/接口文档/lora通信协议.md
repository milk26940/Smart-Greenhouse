# LoRa通信协议

## 1. 说明

本协议用于 STM32 发送端与 LoRa 对端之间的透传字符串通信。

- 上行方向：STM32 周期上报传感器数据和电机状态
- 下行方向：对端通过透传字符串下发电机启停和速度命令
- 编码方式：ASCII 文本
- 默认上报周期：5 秒
- 帧结束：建议以下行换行 `\r` 或 `\n` 作为一帧结束，上行可直接发送完整文本

## 2. 上行协议

### 2.1 帧格式

```text
ENV,T=<temperature>,H=<humidity>,L=<light_raw>,CO2=<co2_ppm>,M=<ON|OFF>,S=<speed_percent>
```

### 2.2 字段说明

- `ENV`：环境数据帧标识
- `T`：温度，单位摄氏度，保留 1 位小数
- `H`：湿度，单位 `%RH`，保留 1 位小数
- `L`：光敏 ADC 原始值，范围 `0~4095`
- `CO2`：二氧化碳浓度，单位 `ppm`
- `M`：电机状态，`ON` 或 `OFF`
- `S`：电机速度百分比，范围 `0~100`

### 2.3 示例

```text
ENV,T=25.3,H=60.1,L=1234,CO2=567,M=ON,S=35
```

## 3. 下行协议

### 3.1 帧格式

```text
CTRL,M=<ON|OFF>,S=<speed_percent>
```

### 3.2 字段说明

- `CTRL`：控制帧标识
- `M`：电机启停命令
  - `ON`：启动电机
  - `OFF`：关闭电机
- `S`：速度百分比，合法范围 `0~100`
- 非 `CTRL` 开头的行会被忽略，不会上报 `CMD:ERR`

### 3.3 示例

启动电机并设置 80% 速度：

```text
CTRL,M=ON,S=80
```

停止电机：

```text
CTRL,M=OFF,S=0
```

## 4. 非法命令处理

- 帧头不是 `CTRL`：忽略
- `M` 不是 `ON/OFF`：忽略
- `S` 超出 `0~100`：忽略
- 空帧或不完整帧：忽略
- 调试串口 `USART1` 会输出 `CMD:OK ...` 或 `CMD:ERR ...` 便于联调

## 5. 透传模式约束

- LoRa 模块初始化仍由 STM32 上电后发送 `+++`、`AT+MODE0`、`AT+LEVEL2`、`AT+CHANNEL01`、`AT+RESET`
- 初始化完成后进入透传模式
- `USART2` 仍作为 LoRa 串口
- `USART1` 仍镜像 `USART2` 的收发内容到 PC
- 下行命令建议发送后带换行，便于 STM32 识别一帧结束

## 6. 当前实现备注

- DHT11 读取失败时，STM32 保留上一次有效温湿度
- CO2 串口数据按 6 字节二进制帧解析：`2C <高字节> <低字节> 03 FF <校验和>`
- 光敏值当前上报 ADC 原始值，换算阈值可在上位机侧完成

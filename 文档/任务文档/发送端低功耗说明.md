# 发送端低功耗说明

## 本次改动

针对 `发送端` STM32F103 + FreeRTOS 工程，已开启基础低功耗：

- 启用 `FreeRTOS tickless idle`
- 在系统空闲时进入 Cortex-M3 `Sleep` 模式
- 通过中断正常唤醒后继续运行任务

涉及文件：

- `发送端/Core/Inc/FreeRTOSConfig.h`
- `发送端/Core/Src/freertos_demo.c`

## 当前方案边界

本次仅实现基础低功耗，不包含以下内容：

- 不进入 `STOP` / `STANDBY`
- 不主动关闭 `USART` / `ADC` / `TIM` / `GPIO` 外设时钟
- 不修改 LoRa、串口转发、传感器采集、执行器控制逻辑

因此本次目标是：

- 降低空闲期 CPU 活动时间
- 保持现有通信和唤醒路径兼容
- 以较低风险引入基础省电能力

## 关键配置

`FreeRTOSConfig.h` 中新增：

- `configUSE_TICKLESS_IDLE 1`
- `configEXPECTED_IDLE_TIME_BEFORE_SLEEP 2`
- `configPRE_SLEEP_PROCESSING`
- `configPOST_SLEEP_PROCESSING`

`freertos_demo.c` 中新增：

- `PRE_SLEEP_PROCESSING()`
- `POST_SLEEP_PROCESSING()`

当前钩子仅保留最小同步屏障，不做外设时钟关闭。

## 触发条件

当所有任务都处于阻塞或延时状态时，FreeRTOS 会判断系统进入空闲窗口，此时触发 tickless idle，并执行睡眠流程。

当前工程中典型空闲来源包括：

- 传感器任务 2 秒周期采样
- LoRa 任务在无下行时延时等待
- 执行器任务阻塞等待队列

## 后续建议

如果后续需要更明显的功耗下降，可以在确认唤醒源和时钟恢复流程后，再评估：

- 关闭部分空闲外设时钟
- 进入 `STOP` 模式
- 增加板级电流实测

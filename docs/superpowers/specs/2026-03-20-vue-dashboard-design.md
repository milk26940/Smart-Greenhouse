# `vue-project` 监控大屏界面优化设计

## 背景

当前 `网页/vue-project` 已实现智慧农业大棚监控页面，具备以下能力：

- 展示后端健康状态、实时环境数据、历史记录、统计数据
- 展示节点在线状态
- 通过浏览器直接连接 MQTT Broker，并显示连接日志与最近消息

现有页面功能完整，但视觉风格偏浅色卡片展示页，监控大屏感不足；实时环境数据虽然存在，但在整体信息结构中不够突出，不利于快速查看关键指标。

## 目标

本次优化聚焦前端界面改版，不调整后端接口和数据结构，目标如下：

- 将页面调整为“科技监控风”的数据大屏
- 将“实时环境指标”提升为页面主视觉中心
- 保留历史趋势、节点状态、MQTT 面板，但降低其视觉优先级
- 在桌面端形成明显的大屏监控感，同时保证移动端可读性
- 修复当前页面中存在的中文显示异常问题

## 非目标

以下内容不在本次范围内：

- 不新增后端接口
- 不修改现有 API 数据结构
- 不引入新的页面路由
- 不对业务逻辑做大规模重构
- 不新增复杂第三方依赖

## 推荐方案

采用“中轴数据大屏”方案。

### 原因

- 最符合“实时环境指标优先”的目标
- 能兼顾现有历史趋势、MQTT、节点状态等模块
- 相比左右偏置布局，信息分布更均衡，桌面端更像监控中心
- 在不拆分页面的前提下，可通过单文件重构完成

## 信息架构

页面结构调整为以下层级：

### 1. 顶部总控区

用于承载页面标题与系统级状态：

- 页面标题：智慧农业大棚监控中心
- 副标题：说明当前页面面向实时监控、链路观察与趋势分析
- 系统状态卡：
  - 后端在线状态
  - 数据存储状态
  - 数据来源
  - 最近更新时间
  - 当前时间或控制提示
- 操作区：
  - 刷新总览
  - 编辑 MQTT 配置

### 2. 主视觉指标区

作为页面核心区域，突出 6 项环境指标：

- 温度
- 湿度
- 土壤湿度
- 光照
- CO2
- 亮度

每张指标卡包含：

- 中文名称
- 大号实时数值
- 单位
- 状态色或发光描边
- 轻量装饰，例如数值条、发光边、角标

### 3. 辅助状态区

放置非主视觉但仍需频繁查看的信息：

- 视觉状态
- 运动检测
- 绿植比例
- 节点在线状态

该区域使用紧凑面板，强调“状态判断”而不是“大号数字”。

### 4. 趋势分析区

保留已有历史数据能力，包含：

- 时间范围选择
- 历史记录表格
- 温度 / 湿度 / 土壤湿度折线图
- 光照 / CO2 折线图

调整重点：

- 视觉层级低于主视觉指标区
- 图表改为深色大屏配色
- 表格改为更紧凑的监控面板样式

### 5. MQTT 通信区

保留现有能力，强调为“链路监控面板”：

- MQTT 连接状态
- 连接 / 断开 / 配置 / 清空日志操作
- 当前连接参数摘要
- 实时日志
- 最近消息预览

该区域视觉上更接近“终端监控窗”，与环境指标区形成层次差异。

## 视觉方向

整体风格选择：科技监控风。

### 视觉特征

- 深色背景作为全局底色
- 青蓝色、荧绿色作为高亮色
- 卡片采用半透明玻璃面板和弱发光边框
- 使用更强对比度的标题和大号数字
- 通过渐变、光晕、网格感背景增强控制舱氛围

### 具体约束

- 避免继续使用当前偏浅色、温和农业展示风
- 避免使用常见泛用后台模板观感
- 保持中文界面为主
- 字体仍优先使用本地常见中文字体栈，避免因远程字体加载影响可用性

## 交互设计

### 加载与刷新

- 页面加载时分层出现：
  - 顶部总控区
  - 实时指标区
  - 趋势与 MQTT 区
- 刷新按钮继续调用现有 `loadDashboard`
- 加载中状态在按钮和局部区域有明确反馈

### 状态反馈

- 后端在线 / 离线使用明确色彩区分
- MQTT 状态徽标按照当前连接态展示不同颜色与动画
- 错误横幅继续保留，但与深色主题匹配

### 弹窗

- MQTT 配置弹窗保留现有表单结构
- 仅做视觉重构，不改字段含义和提交流程

## 数据与逻辑边界

本次允许的逻辑调整：

- 新增少量前端展示型计算属性
- 增加状态映射函数，用于颜色、等级、文案展示
- 修正中文文案编码问题

本次不调整：

- `fetchJson` 请求机制
- `loadDashboard` 并发加载流程
- MQTT 连接核心逻辑
- 后端入库请求结构

## 实现策略

以最小改动为原则，优先在 [`App.vue`](d:/STM32Project/智慧农业大棚/网页/vue-project/src/App.vue) 内完成：

- 重组模板结构
- 整理文本常量和状态文案
- 增加少量展示辅助函数
- 重写 `scoped style`

尽量不拆分组件，避免本次任务范围膨胀；若实现过程中发现单文件过长，可只做有限度的局部抽离，但不是默认方案。

## 测试与验证

完成后至少执行以下验证：

- `npm run build`
- 若测试基线可用，执行 `npm run test:unit -- --run`

人工检查重点：

- 页面在桌面端是否形成明确的大屏主次结构
- 实时环境指标是否显著成为视觉中心
- 历史图表、MQTT、节点状态仍能正常查看
- MQTT 配置弹窗可正常打开和关闭
- 中文文案是否恢复正常显示
- 移动端布局是否退化为可读的单列结构

## 风险

- 当前 `App.vue` 内容较多，重构时容易引入模板或样式回归
- 中文乱码如果来自文件编码历史问题，需要在修改时整体修正相关文案
- 深色主题下图表配色需要同步调整，否则可读性会下降

## 验收标准

满足以下条件即视为本次界面优化完成：

- 页面整体视觉风格切换为科技监控风
- 实时环境指标成为首屏主视觉中心
- 现有核心功能未丢失
- 中文显示正常
- 构建通过
